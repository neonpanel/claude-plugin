# Diagnose Lost Batches Cogs — Canonical Claims

These are verified facts. Only make claims that appear in this list.

- NeonPanel tracks inventory by SKU + Country Code. Units received under one country code (e.g., US) are invisible to the FIFO queue for a different country code (e.g., UK), even if they are in the same physical warehouse.
- An Identity Mismatch occurs when items are received into a warehouse with the wrong country code selected on the inbound document (PO or Shipment). The result is that sales in the correct marketplace find an empty FIFO queue, producing Lost Batches despite the warehouse physically holding stock. The signal is a high-value PO where the Landed Cost report shows $0.00 for the expected SKUs in that warehouse.
- A Hollow Inbound occurs when a shipment is recorded as Received and assigned a batch ID, but the item_landed_cost is $0.00. This happens when the shipment was created independently (e.g., from an Amazon STA pull) and was never connected to a Purchase Order or Expense Bill. The fix is to link the hollow shipment to its source PO or Bill so cost propagates forward to sales.
- For outbound transfers and internal shipments, the Origin Warehouse must be a physical asset warehouse (e.g., Main-UK) that has previously received stock. Shipping out of a Transfer warehouse or Lost & Found warehouse will produce lost batches because those warehouses have no FIFO inventory to debit.
- Diagnostic rule of thumb for lost COGS: if the quantity is correct but COGS is $0, check the connections (PO-to-Shipment link, Bill attachment). If COGS is $0 and the units are in Lost & Found, check the identity (Country Code on the inbound document or Warehouse name).
- The Warehouse Fork is the first diagnostic signal: if the problematic transaction shows warehouse = Lost & Found, it is a Physical Gap (stock never arrived in the selling warehouse ledger). If the warehouse is an actual location (e.g., Main-UK), it is a Cost or Identity Gap (stock exists but the FIFO engine cannot see it due to $0 cost or wrong identity).
- If a Receipt (inbound) and a Shipment (outbound) occur on the same day for the same SKU and warehouse, the system may process the outbound first, causing a lost batch. To prevent this, ensure inbound document dates always precede outbound document dates.
- The Landed Cost Analysis Report in NeonPanel verifies whether the dollar value recorded on a Purchase Order actually made it into the warehouse ledger. Use it when a PO shows high value but the warehouse shows $0 for the expected SKUs.
- The Inventory Audit Report in NeonPanel compares the computed ledger balance against the Warehouse Balances view. Use it to find quantity discrepancies between what the system calculated and what the warehouse balance displays.
- A PO showing high value ($10k+) but the Landed Cost report showing $0.00 for expected SKUs is the primary signal of an Identity Mismatch or Hollow Inbound. Always check the Country Code on the inbound document first, then check whether the shipment is linked to a PO or Bill.
