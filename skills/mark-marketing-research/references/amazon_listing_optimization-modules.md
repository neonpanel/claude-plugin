# Amazon Listing Optimization — Module Map

## keyword-research-seo

Keyword research fundamentals, buyer intent analysis, Amazon search algorithm behavior (A10, Rufus, COSMO), and backend search term optimization. Covers primary/secondary/long-tail keywords, search term tools, intent-over-volume strategy, Generic keyword field best practices, prohibited search terms, byte limits, and bulk search term updates.

**Common user intents:**
- How do I find the right keywords?
- What tools should I use for keyword research?
- How does Amazon's search algorithm work?
- What is buyer intent and why does it matter?
- How do I optimize backend search terms?
- What words are prohibited in search terms?
- How do I update search terms in bulk?
- Why aren't my search terms showing up?
- What's the byte limit for search terms?
- Is the Generic keyword field required for my product type?

**Common confusions:**
- Thinking high volume always means high relevance
- Stuffing keywords instead of focusing on intent
- Not understanding that Amazon AI reads context, not just exact phrases
- Repeating front-end keywords in backend search terms (wastes space)
- Confusing bytes with characters (umlauts and special characters use 2+ bytes)
- Including brand names or ASINs in search terms (prohibited)
- Using punctuation or articles in search terms (wastes byte space)
- Not knowing the Generic keyword field is required for 23 product types since December 2023

**Artifacts to request:**
- Product category
- Competitor ASINs
- Current keyword list
- Current backend search terms

**Key objects:** Primary keywords, Secondary keywords, Long-tail keywords, Buyer intent, Amazon A10 algorithm, Rufus AI, COSMO, Search term tools (Helium 10, Jungle Scout, SellerSprite), Generic keyword field, Backend search terms, Byte limit, Prohibited search terms, Category Listing report, Bulk upload
