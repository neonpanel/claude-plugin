# Support — Playbook

# Support Playbooks\n\n1. **Access denied** (`sup-access-issues`)\n   - Symptom: User can't access workspace\n   - Checks: user email, role, workspace ID\n   - Outcome: request role confirmation or escalate\n\n2. **Login error** (`sup-access-issues`)\n   - Symptom: Login failure\n   - Checks: error message, timestamp\n   - Outcome: collect details\n\n3. **Unexpected error** (`sup-errors`)\n   - Symptom: Error message\n   - Checks: screenshots, repro steps\n   - Outcome: collect artifacts and escalate if widespread\n\n4. **Open ticket** (`sup-escalation`)\n   - Symptom: User requests escalation\n   - Checks: impact and urgency\n   - Outcome: collect ticket details\n\n5. **Usage question** (`sup-usage-questions`)\n   - Symptom: How do I use feature\n   - Checks: feature scope\n   - Outcome: provide high-level guidance\n\n6. **Billing issue** (`sup-account-changes`)\n   - Symptom: Billing question\n   - Checks: workspace, account owner\n   - Outcome: escalate to finance/admin\n\n7. **Permissions confusion** (`sup-access-issues`)\n   - Symptom: Role confusion\n   - Checks: desired actions\n   - Outcome: request role definitions\n\n8. **System outage** (`sup-errors`)\n   - Symptom: Multiple users affected\n   - Checks: timeframe, scope\n   - Outcome: escalate immediately\n\n9. **Settings change request** (`sup-account-changes`)\n   - Symptom: Change settings\n   - Checks: setting name\n   - Outcome: request identifiers\n\n10. **Security concern** (`sup-escalation`)\n    - Symptom: Security incident\n    - Checks: impact\n    - Outcome: escalate immediately\n\n## Provide Report Links\n\n1. User asks for a report link, report URL, or how to access a report.\n2. Call the `neonpanel_listReports` tool to retrieve available reports.\n3. From the tool response, extract the report **group** and **slug** for each relevant report.\n4. Construct the full clickable URL using the pattern: `https://my.neonpanel.com/app/reports/{group}/{slug}`\n5. Present the report name alongside its clickable link.\n6. If the user asked for a specific report, filter results and provide only the matching link(s).\n7. If the tool returns an error or no results, explain the issue and offer to retry or escalate.\n8. **Never refuse to provide report links.** The tool provides sufficient data to construct URLs. Do not claim otherwise.
